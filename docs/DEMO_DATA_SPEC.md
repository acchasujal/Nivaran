# DEMO_DATA_SPEC.md
**Defines the exact data the seeding script must produce. The seeded state is the baseline for every demo run. Every field is specified to prevent the seeding script from inventing data.**

---

## Required Demo State

```
2 clusters
6 issues (3 per cluster)
2 impact summaries (1 per cluster, pre-generated)
6 action drafts (3 per cluster: complaint, rti, community_summary)
0 escalations (these are generated live)
```

## Photo Asset Requirements

| Filename | Issue Type | Requirements |
|---|---|---|
| `pothole_01.jpg` | road_damage | Clear daytime photo, pothole visible, Indian urban context |
| `pothole_02.jpg` | road_damage | Slightly different angle/location; same cluster |
| `pothole_03.jpg` | road_damage | Most severe-looking; severity 5 |
| `streetlight_01.jpg` | lighting | Nighttime or low-light, non-functional light visible |
| `streetlight_02.jpg` | lighting | Multiple lights in frame if possible |
| `streetlight_03.jpg` | lighting | Similar location, different angle |

**Photo rules:**
- All photos must be real photos, not AI-generated or stock imagery.
- Photos must not contain identifiable faces.
- Photos must be plausibly from Mumbai or a comparable Indian urban context.
- File size: max 2MB each (pre-compressed for seeding speed).
- Format: JPG.

**If real photos are unavailable:** Use Creative Commons licensed photos of Indian civic infrastructure. Document the source in `scripts/demo_assets/SOURCES.md`. Never use stock photo watermarked images.

---

## Seeding Script Requirements

The script at `scripts/seed_demo.py` must:

1. Accept a `--wipe` flag that deletes all rows from all tables before seeding.
2. Insert all records above in dependency order (clusters → issues → impact_summaries → action_drafts).
3. Use the exact IDs specified above (`demo-cluster-001`, `demo-issue-001`, etc.) for predictable URLs.
4. Print a confirmation summary:
   ```
   ✓ 2 clusters created
   ✓ 6 issues created
   ✓ 2 impact summaries created
   ✓ 6 action drafts created (status=pending_review)
   ✓ Demo state ready
   ```
5. Run in under 5 seconds.
6. Be idempotent: running twice with `--wipe` produces the same state.
